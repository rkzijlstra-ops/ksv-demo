Let op de valkuil: een feature-branch pushen landt op kluslus-test alleen als preview (eigen URL achter Vercel-login), niet vanzelf op het publieke kluslus-test.vercel.app. Daarvoor moet je de deployment promoten. Eén merge naar master deployt prod én demo tegelijk; het enige verschil tussen die twee is de env (DEMO_MODE + eigen DB).

5 · Per dienst: wat het doet & naslag bij storing

Klik open. Telkens dezelfde drie regels: wat het doet, hoe het koppelt, en wat je doet als het stuk is.

▸ GitHub code + geschiedenis
Wat
De opslag van alle code en de geschiedenis. Repo: github.com/rkzijlstra-ops/ksv-demo.
Koppeling
Elke push/PR start de CI (Actions). Een push naar master laat Vercel prod + demo opnieuw deployen.
Bij storing
CI rood? Open het Actions-tabblad in de repo en lees de gefaalde stap. Vaak: schema-drift (migratie niet op alle DB's) of typecheck.
▸ Vercel host de app · 3 projecten
Wat
Bouwt en host de app. Drie projecten: prod kluslus-productie (mijn.kluslus.nl, default kluslus-productie.vercel.app), demo kluslus-demo, test kluslus-test. Team bkm-s-projects.
Koppeling
master → prod + demo. Branch → preview. Cron-jobs draaien hier. Alle keys staan als Environment Variables per project. De PWA-service-worker krijgt zijn versie automatisch uit het git-commit (VERCEL_GIT_COMMIT_SHA) via een prebuild-script (scripts/genereer-sw-versie.mjs); geen handmatige VERSION-bump meer.
Bij storing
Deploy faalt → Deployments-log. Iets werkt niet maar build is groen → meestal een ontbrekende of foute env-var (Project Settings → Environment Variables). Feature staat niet op publieke test-URL → promoten (zie OMGEVINGEN.md).
▸ Supabase database + auth · 3 DB's
Wat
De databases en het inloggen (Google + magic-link). Drie losse DB's: prod qbynjfscdxhwdkzfqjjg, test mslexkwwhhwlxmwbnyff, demo bcaallhweqamkrxtlwvx.
Koppeling
Elke Vercel-omgeving wijst via env naar zijn eigen DB. prod-migraties doe jij met de hand; test + demo via npm run migrate:test.
Bij storing
CI rood na een schema-wijziging → je hebt de migratie niet op alle drie gedraaid (schema-drift). Regel: elke migratie op prod (hand) én test+demo (script).
▸ Resend uitgaande mail · via AWS SES
Wat
Verstuurt de mails uit de app (rapport, melding) én ontvangt de inbound-mail (Resend Receiving). Draait onder water op AWS SES; DKIM via resend._domainkey.
Koppeling
Key RESEND_API_KEY in Vercel. RESEND_FROM / RESEND_REPLY_TO bepalen afzender en antwoordadres. Grendel: MAIL_DRY_RUN + MAIL_ALLOWLIST. Inbound via twee Receiving-webhooks: prod → https://mijn.kluslus.nl/api/inbound, test → de kluslus-test-app (INBOUND_DOMAIN=klus-test.kluslus.nl).
Bij storing
Mail komt niet aan → resend.com → Logs. Let op de strenge DMARC p=reject: afzender en SPF/DKIM moeten uitlijnen. Reply-To staat per monteur ingesteld.
▸ CM.com uitgaande SMS · LIVE
Wat
Verstuurt SMS-berichten. Live in productie sinds 2026-06-10 via gw.cmtelecom.com.
Koppeling
CM_PRODUCT_TOKEN + CM_GW_URL + SMS_AFZENDER in Vercel. Grendel: SMS_DRY_RUN + SMS_ALLOWLIST.
Bij storing
SMS komt niet aan → check of SMS_DRY_RUN=1 (stilgezet) of de allowlist te smal staat. Token in het CM-dashboard.
▸ Google Workspace menselijke mailboxen
Wat
Je gewone mailboxen op het domein: info@kluslus.nl + aliassen antwoord@ / planning@ / support@ (sinds 2026-06-21).
Koppeling
De MX van het hoofddomein (@) wijst naar Google. Dit staat los van de app-inbound op klus..
Bij storing
Mailbox-gedoe → admin.google.com. Verwar dit nooit met de app-inbound; die loopt via klus.kluslus.nl → AWS SES.
▸ Vimexx domein + DNS
Wat
Registrar van kluslus.nl en de plek waar alle DNS-records staan (nameservers = zxcs).
Koppeling
CNAME mijn → Vercel (de app), MX @ → Google (mailbox), MX klus/klus-test/send → AWS SES (app-mail in/uit, prod + test). SPF/DKIM/DMARC als TXT.
Bij storing
DNS aanpassen in het Vimexx-paneel (DirectAdmin). De klus.-, klus-test.- en send.-MX niet aanraken, anders breekt app-mail.
▸ Anthropic / OpenAI AI in de app
Wat
De AI-modellen die de app gebruikt voor tekstfuncties.
Koppeling
ANTHROPIC_API_KEY / OPENAI_API_KEY in Vercel. In CI staan dummy-keys (de e2e roept de AI niet echt aan).
Bij storing
AI-functie faalt → controleer de key in Vercel env en het tegoed in het betreffende dashboard.
▸ Vercel Cron geplande taken
Wat
Geplande aanroep van de app. Nu: /api/cron/bevestig-herinneringen, schema 0 7,14 * * * (07:00 + 14:00).
Koppeling
Staat in vercel.json, beveiligd met CRON_SECRET. Was Hobby (max 1×/dag), sinds Pro 2×/dag.
Bij storing
Cron loopt niet → Vercel-project → Crons-tab voor de laatste runs; check dat het project op het Pro-plan staat.
